# Tracking Money

개인 보유종목, 관심종목, 미국 선행 지표, 뉴스 이슈, 캘린더를 한 화면에서 보는 투자 대시보드입니다.

## 주요 기능

- 내일 국내장 방향점수
- 기준/상방/하방 내일장 예측 시나리오
- 보유종목과 관심종목 영향도
- NQ 선물, SOX, VIX, 달러/원, 미국 금리 등 선행 지표
- Yahoo Finance chart 기반 무료 시세 프록시
- 네이버 뉴스 API 기반 사용자 키워드 뉴스 피드
- 보유/관심종목 기반 이벤트 캘린더 API
- OpenDART 기반 국내 공시/실적 원문 체크
- 보유종목/관심종목 브라우저 저장 및 편집
- 가격, 등락률, 뉴스 키워드, 방향점수 기반 사용자 알림 규칙
- 지표, 뉴스, 캘린더, 관심가를 합친 오늘 액션 큐
- 장전 계획, 리스크 대응, 장후 리뷰를 저장하는 투자 저널
- 실적, 매크로, 정책 이벤트 캘린더

## 실시간 데이터 연결

앱은 Vercel 서버 함수를 통해 외부 데이터를 가져옵니다. 브라우저가 직접 외부 API를 호출하지 않습니다.

```text
/api/news    네이버 뉴스 API 프록시
/api/quotes  Yahoo Finance chart 기반 시세 프록시
/api/calendar  보유/관심종목 기반 매크로·실적 이벤트 캘린더
/api/disclosures  OpenDART 기반 국내 공시 프록시
/api/health  데이터 소스와 환경변수 연결 상태 진단
/api/profile  선택형 서버 프로필 동기화
```

`/api/quotes`는 아래 기본 심볼을 수집합니다.

```text
005930.KS  삼성전자
000660.KS  SK하이닉스
035420.KS  NAVER
373220.KS  LG에너지솔루션
AAPL       Apple
TSLA       Tesla
NVDA       NVIDIA
NQ=F       나스닥100 선물
ES=F       S&P500 선물
^SOX       필라델피아 반도체지수
^VIX       변동성 지수
DX-Y.NYB   달러 인덱스
^TNX       미국 10년물 금리
KRW=X      달러/원
```

시세 수집이 실패하면 기존 샘플 데이터로 자동 fallback됩니다.

보유종목, 관심종목, 뉴스 키워드, 사용자 알림 규칙은 화면에서 직접 추가, 수정, 삭제할 수 있습니다. 입력값은 브라우저 `localStorage`에 저장되며, 새로 추가한 국내 6자리 종목과 미국 티커도 `/api/quotes` 수집 대상에 포함됩니다. 뉴스 키워드는 `/api/news` 수집 파라미터와 뉴스 화면 필터에 바로 반영됩니다.

`/api/calendar`는 현재 추적 중인 심볼을 기준으로 국내장 개장 전 선행지표 점검, 주요 매크로 이벤트, 정책 이벤트, 종목별 실적 발표 구간을 생성합니다. `watch`는 매일 확인할 체크리스트, `estimated`는 공식 일정으로 재확인이 필요한 추정 이벤트입니다.

`/api/disclosures`는 OpenDART의 공시검색 API를 서버에서 호출해 국내 보유/관심종목의 최근 공시 원문 링크, 중요도, 예상 영향을 정리합니다. 현재 기본 매핑은 삼성전자, SK하이닉스, NAVER, LG에너지솔루션을 포함합니다.

예측 화면은 방향점수, 선행 지표, 뉴스, DART 공시, 캘린더, 보유종목 비중을 룰 기반으로 합쳐 기준/상방/하방 시나리오, 예상 출발 범위, 핵심 변수, 장전 체크리스트를 보여줍니다.

대시보드와 알림 화면의 액션 큐는 방향점수, 선행 지표, 보유종목 비중/등락, 관심가 근접, 네이버 뉴스 중요도, DART 공시, 캘린더 이벤트를 합쳐 오늘 먼저 확인할 항목을 정렬합니다. 알림 화면에서는 가격 이상/이하, 등락률 이상/이하, 뉴스 키워드, 방향점수 이상/이하 조건을 직접 만들고 현재 발동 여부를 확인할 수 있습니다.

투자노트 화면에서는 오늘 액션 큐를 체크리스트로 처리하고, 장전 계획/리스크 대응/장후 리뷰를 브라우저에 자동 저장합니다. 날짜가 바뀌면 새 저널이 시작됩니다.

설정 화면의 `실데이터 수신 상태`와 `서버 연결 진단`에서는 현재 화면이 실제로 받은 시세/뉴스/공시/캘린더 상태, Vercel 환경변수 설정 여부, 아직 예정 단계인 데이터 소스를 확인할 수 있습니다. `뉴스 키워드 관리`와 `데이터 백업`에서는 개인 뉴스 키워드와 알림 규칙까지 JSON으로 백업/복원할 수 있습니다.

`서버 동기화`는 Upstash Redis REST 호환 저장소를 선택적으로 사용합니다. 환경변수를 설정하면 보유종목, 관심종목, 뉴스 키워드, 알림 규칙, 투자노트를 서버에 저장하고 다른 기기에서 불러올 수 있습니다. 동기화 키는 화면에서 입력한 요청에만 사용하고 브라우저에 저장하지 않습니다.

## API 환경변수

Vercel 프로젝트의 `Settings > Environment Variables`에 아래 값을 추가합니다.

```text
NAVER_CLIENT_ID=네이버_개발자센터_Client_ID
NAVER_CLIENT_SECRET=네이버_개발자센터_Client_Secret
OPENDART_API_KEY=오픈다트_API_인증키
KV_REST_API_URL=Upstash_Redis_REST_URL
KV_REST_API_TOKEN=Upstash_Redis_REST_TOKEN
PROFILE_SYNC_KEY=직접_정한_개인_동기화_키
```

로컬에서 Vercel 서버 함수까지 테스트할 때는 `.env.local`에 같은 이름으로 넣고 Vercel CLI로 실행합니다. `.env.local`은 커밋하지 않습니다.
